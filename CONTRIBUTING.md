# Contributing to Quantified Self

Welcome, Commander! We're thrilled you want to help optimize the mission. Whether you're fixing a bug, adding a new feature, or improving the documentation, your contributions are highly valued.

## How to Contribute

### 1. Reporting Bugs
- Use the GitHub Issue Tracker.
- Provide a clear and descriptive title.
- Include steps to reproduce the issue.
- Describe the expected vs. actual behavior.

### 2. Suggesting Features
- Open an Issue with 'Enhancement' tag.
- Explain why the feature would be useful for the mission.
- Provide mockups or design ideas if possible.

### 3. Submitting Pull Requests
1. Fork the repository.
2. Create a new branch: `git checkout -b feature/your-feature-name`.
3. Make your changes and commit them: `git commit -m "feat: add mission objective x"`.
4. Push to your fork: `git push origin feature/your-feature-name`.
5. Open a Pull Request against the `main` branch.

## Code of Conduct
Please be respectful and collaborative. We aim to build an inclusive environment for all commanders.
